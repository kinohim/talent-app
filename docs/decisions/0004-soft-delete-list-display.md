# 0004: 論理削除データの一覧表示方法

## 状況

全テーブル共通で`deletedAt`による論理削除を採用しているが（`docs/design/data-model.md`）、一覧画面（社員一覧、各種マスタ一覧）で削除済みデータをどう扱うかが未確定❓だった。「常に非表示」と「フィルタで表示切替」の2案があった。

## 決定

- 一覧画面はデフォルトで削除済みデータを**非表示**にする
- 「削除済みを表示」のようなフィルタ切替をUIに用意し、必要な時だけ表示できるようにする

## 比較・理由

| 観点 | 常に非表示 | フィルタ切替（採用） |
|---|---|---|
| 実装コスト | 低い（`where: { deletedAt: null }`固定） | 中（一覧画面にフィルタUI、APIにクエリパラメータ追加） |
| 運用性 | 誤削除の確認・復元がしづらい | 削除済みデータを確認・復元しやすい |
| ハッカソンでのデモ映え | 論理削除の仕組みが見えない | 論理削除→復元の一連の流れをデモで見せられる |

社内ツールとして誤操作からの復元導線があった方が実用的であること、また「削除しても消えない」という論理削除の価値をハッカソンのデモで見せやすいことから、実装コストの増加を許容してフィルタ切替を採用した。

## 影響

- 一覧系API（社員一覧、各マスタ一覧）に`includeDeleted`等のクエリパラメータを追加する（`docs/design/api-design.md`で定義）
- 一覧画面（S-09、S-12〜S-16、S-18等）のUIに表示切替コントロールを追加する
- 削除済みデータの復元（`deletedAt`を`null`に戻す）機能もあわせて検討するとよい（フェーズ2以降でも可）
