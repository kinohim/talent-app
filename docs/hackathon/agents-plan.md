# サブエージェント設計方針

ハッカソンの「ハーネス」評価軸に対応。フェーズごとに役割を分けたサブエージェントを想定する。

| エージェント名 | 役割 |
|---|---|
| requirements-agent | 要件整理・仕様策定 |
| design-agent | アーキテクチャ設計・データモデル設計 |
| impl-agent | 実装・リファクタリング・デバッグ |
| test-agent | テスト戦略・ケース設計・自動テスト生成 |
| infra-agent | DB構築・デプロイ・監視 |
| design-review-agent | 設計書（要件定義書・基本設計書・詳細設計書・DB設計書・画面設計、およびExcel出力版）のレビュー。矛盾・抜け漏れ・ADRとの不整合を洗い出し、報告のみ行う（資料への反映はユーザー承認後） |

Claude Codeで開発する際は、上記の定義を `.claude/agents/` 配下にエージェントごとのMarkdownファイルとして配置して運用する。
