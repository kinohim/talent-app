# AI駆動開発ロードマップ

ハッカソンの評価軸（AI観点50点）は、そのままAI駆動開発の工程と対応している。この順番で進める。

| フェーズ | 内容 | 主なAI活用 |
|---|---|---|
| 1. 設計工程 | 要件整理・仕様策定・アーキテクチャ設計・データモデル設計 | 要件をAIと壁打ちして整理、構成案・ER図・API設計をAIに提案させ比較検討 |
| 2. 実装工程 | コーディング | CLAUDE.mdでプロジェクト文脈を共有し、コード生成・リファクタ・デバッグをAIに任せる |
| 3. テスト工程 | テスト戦略・ケース設計・自動テスト生成 | テスト観点の洗い出しと自動テストコード生成をAIと行う |
| 4. インフラ | DB構築・デプロイ・監視 | 構築手順・IaC・監視設定をAIと一緒に組む |
| 5. ハーネス | CLAUDE.md／サブエージェント／Skills／MCPなど横断基盤 | 上記すべてを支える仕組み自体を整備・改善する |

各フェーズで「使ったプロンプト」「AIの提案」「採用/不採用の判断」を `docs/decisions/` と `ai-usage-log.md` に残す。これが最終発表資料の元ネタになる。

## 進め方

1. まず要件定義書を `docs/requirements/` に配置し、内容を確認する
2. 設計工程（アーキテクチャ・データモデル）をAIと詰める
3. 実装をサブエージェント分担（`agents-plan.md` 参照）で進める
4. テスト・インフラを構築する
5. 並行してハーネス（CLAUDE.md・エージェント定義）を継続的に改善する
6. 最後にここまでの記録を `slides/` のパワポにまとめる
