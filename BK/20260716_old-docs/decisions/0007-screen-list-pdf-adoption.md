# 0007: 画面体系を「01_画面一覧.pdf」に準拠させる

## 状況

これまで設計書の画面体系は`docs/design/screens.md`のS-01〜S-20（社員ID＋パスワード認証、パスワードリセット画面、Excel取込画面等を含む）で管理してきた。その後チームで画面一覧を再整理し、`docs/design/excel-export/01_画面一覧.pdf`（22画面：AUTH001／REF001〜REF008／EDT001〜EDT007／MST001〜MST005／CMN001）が作成された。

## 決定

- **画面は「01_画面一覧.pdf」を正とする**（ユーザー決定事項、2026-07-08）。設計書全体（要件定義書・基本設計書・詳細設計書・API設計書・画面設計・データモデル）をこの画面体系に合わせて改訂する
- PDFの記載に伴い、以下も設計に反映する
  - **認証はAzure AD（Microsoft Entra ID）へのリダイレクトログイン**とする（AUTH001）。Auth.jsは継続利用し、プロバイダをCredentialsからMicrosoft Entra IDに変更する。パスワード管理（ハッシュ保存・変更・リセット画面/API）は廃止
  - **組織は「事業部＞部署＞Gr」の3階層**で管理する（MST004）。`Department`テーブルに親子関係と階層区分を追加
  - **現場（site）マスタを新設**する（MST005）。プロジェクト経歴の現場は自由記述からマスタ参照に変更
  - **印刷用プレビュー（PDF出力）画面を追加**する（REF005）。個人の経歴書単位のPDF出力であり、一覧のCSV/Excel出力を持たない決定（ADR `0006`）とは対象が異なる
  - **スキルマップ／組織ダッシュボード（REF008）を追加**する。旧「資格保有者一覧（S-11）」「スキル保有者一覧（S-17）」の横断検索要件は、経歴書一覧（REF002）の検索条件とREF008の集計表示に統合する
  - **編集画面はセクション単位に分割**する（EDT001基本情報／EDT002経歴概要・自己PR／EDT003スキル／EDT004資格／EDT005プロジェクト経歴）
  - **アカウント管理はEDT006（新規登録：社員ID・メールアドレス＝Azure AD照合キー・所属部署・初期権限の事前登録）とEDT007（部署・権限変更、退職・無効化）**で行う

## 旧画面体系からの主な削除・変更

| 旧（S-xx） | 新体系での扱い |
|---|---|
| S-02〜S-04 パスワードリセット申請／新規登録／変更 | 廃止（Azure AD認証に移行するため不要） |
| S-11 資格保有者一覧／S-17 スキル保有者一覧 | REF002（経歴書一覧の検索）とREF008（ダッシュボード）に統合 |
| S-13 駅・路線マスタ管理 | 画面としては廃止。駅・路線データは初期データ投入スクリプトで整備する（保守画面が必要になれば再検討❓） |
| S-20 Excel取込 | 画面としては廃止。初回データ投入はスクリプト直接投入に一本化（ADR `0005`の「またはスクリプトで直接投入する」運用を採用） |
| S-08/S-19 プロフィール編集（1画面一括） | EDT001〜EDT005のセクション別編集画面に分割 |

## 影響

- `docs/design/screens.md`: 画面一覧を22画面構成に全面改訂
- `docs/requirements/requirements.md`: 3.1認証・権限、3.3横断検索、3.4マスタ管理、3.5入出力、フェーズ表を更新
- `docs/design/basic-design.md` / `detailed-design.md` / `api-design.md`: 認証方式・画面参照・エンドポイント構成を更新
- `docs/design/data-model.md` / `db-design.md`: `User`（パスワード関連列の廃止、email必須化、状態列追加）、`Department`（階層化）、`Site`（新設）、`Project`（`site_id`参照化）を更新
- `CLAUDE.md`: 認証の記載をCredentials ProviderからMicrosoft Entra IDプロバイダに更新
- ADR `0001`（技術スタック）: Auth.js採用自体は維持、プロバイダのみ変更
- **既存実装（2026-07-06にNextAuth Credentialsで実装済み）とは乖離が発生する。**実装側の追従は別途impl-agentへの依頼として計画する
- 閲覧権限の変更は本ADRではなくADR `0008-view-permission-department-scope.md`を参照
